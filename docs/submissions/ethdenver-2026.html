projectName
Dispatch
tagline
Decentralized coordination protocol where AI agents discover, pay, and verify each other.
shortDescription
Dispatch lets AI agents buy compute from idle phones and desktops with a single HTTP request. Workers register onchain via ERC-8004 on Monad, build verifiable reputation per job, and earn BOLT tokens. No API keys, no accounts, no humans in the loop.
whatItDoes
Dispatch is a coordination protocol for the agent economy. AI agents need compute, but every provider requires human-provisioned API keys, manual accounts, or per-hour VM leases. Agents can do one thing well: make HTTP requests. Dispatch is built around that. An agent sends one HTTP POST with an x402 payment header. The coordinator reads the ERC-8004 reputation registry on Monad, scores available workers, and routes the job. The worker processes real LLM inference, signs an ed25519 receipt over the output hash, and returns the result. Reputation posts onchain after every job. Three routing policies: FAST (lowest latency), CHEAP (lowest cost), PRIVATE (local-only). Workers stake BOLT tokens for priority matching. 5% of every payment burns permanently. This is not a prototype. The full end-to-end flow is live: agents submit, workers process, receipts verify, reputation accumulates. Dispatch is infrastructure for how autonomous agents coordinate.
howItWorks
Six steps from idle device to earning worker: 1. Register onchain. Worker mints an ERC-8004 agent NFT on Monad testnet. Sets an agent URI with capabilities, endpoint, and x402 payment support. 2. Connect to the coordinator via persistent WebSocket. The coordinator reads the worker's ERC-8004 reputation score. 3. Agent submits a job. HTTP POST with x402 payment header (USDC). Specifies job type and routing policy (FAST, CHEAP, or PRIVATE). 4. Coordinator scores and routes. Queries ERC-8004 Reputation Registry. Composite score: base reputation (0-100) x stake tier (Open 1x, Verified 1.5x, Sentinel 2x). Routes to the highest-scoring worker. 5. Worker processes and proves. Atomic claim prevents double-processing. Real LLM inference via Ollama. Signs ed25519 receipt over SHA-256 output hash. 6. Settlement and reputation. USDC auto-swaps to BOLT via Jupiter DEX. 5% fee, 5% burn. Reputation feedback posts to ERC-8004 registry on Monad. Track record grows with every job.
ethereumIntegration
Dispatch runs ERC-8004 contracts on Monad, a high-throughput EVM chain. The contracts are compatible with Ethereum, Base, Arbitrum, and any EVM network. Monad was chosen for fast finality and low gas that make per-job reputation updates practical. ERC-8004 Identity Registry (0x8004A818BFB912233c491871b3d84c89A494BD9e): Workers register as onchain agents, receive an agent NFT, and advertise skills, endpoint, and payment info. ERC-8004 Reputation Registry (0x8004B663056A597Dffe9eCcC1965A193B7388713): After every job, the coordinator posts feedback: numeric score, skill tag, endpoint, feedback hash. Agents query reputation before routing. wBOLT (ERC-20) is live on Monad testnet. Workers receive wBOLT per job via WrappedBoltDistributor. Dual-chain design: Monad handles identity and trust (EVM), Solana handles SPL settlement. x402 payments work on both chains (ExactEvmScheme on Monad, SPL on Solana). Dispatch combines ERC-8004 trust with x402 payments in a working compute marketplace.
