tagline
Oracle-priced compute: Chainlink feeds set the cost of every AI job in real time.
shortDescription
Dispatch is an autonomous compute marketplace where AI agents buy inference from idle phones and desktops via HTTP. Chainlink CRE price feeds drive dynamic job pricing: real-time USDC/BOLT conversion, demand-responsive rate adjustments, and oracle-verified settlement.
whatItDoes
Dispatch is a decentralized compute marketplace built for AI agents. No accounts, no API keys. An agent sends one HTTP request with an x402 payment header. The coordinator routes the job to the best available worker (idle phone or desktop). The worker runs real LLM inference, signs an ed25519 receipt over the output, and gets paid in BOLT. The problem: compute pricing today is static. Providers set fixed rates regardless of network load, token volatility, or real-time demand. When BOLT fluctuates against USDC, workers and agents absorb the difference. Chainlink CRE fixes this. Oracle price feeds provide real-time USDC/BOLT exchange rates at the moment of settlement. The coordinator reads live demand signals and adjusts job pricing per request. Workers always receive fair market value. Agents always pay current rates. No stale prices, no manual updates.
chainlinkIntegration
Chainlink CRE integrates at three points in Dispatch settlement: 1. Price Feed (USDC/BOLT Conversion) Agents pay USDC via x402 headers, workers earn BOLT. The coordinator queries a CRE price feed for the live USDC/BOLT rate at settlement. Replaces fixed-rate conversion, eliminates stale pricing. 2. Demand-Based Dynamic Pricing A CRE workflow monitors active worker count, queue depth, and completion time. When demand spikes, it outputs a price multiplier applied to the base job rate. Higher prices attract more workers online. Supply and demand balance automatically. 3. Oracle-Verified Settlement Each job produces an ed25519 receipt. CRE verifies the receipt, confirms feed freshness, and triggers BOLT distribution. Result: oracle-attested settlement where price, completion, and payment are all verified. Note: CRE integration is a design proposal. Core Dispatch infrastructure (coordinator, workers, x402 payments, receipts) is live on testnet today.
howItWorks
End-to-end flow with Chainlink CRE: 1. Agent submits job via HTTP POST with x402 payment header (USDC). No SDK, no API key, no wallet required. 2. Coordinator queries the CRE price feed for current USDC/BOLT rate. Job cost = base rate x demand multiplier x exchange rate. 3. Coordinator scores workers by ERC-8004 reputation (onchain on Monad) and stake tier. Routes to highest-scoring worker. 4. Worker runs the job (LLM inference via Ollama on desktop, or text processing on mobile). Signs an ed25519 receipt over the SHA-256 output hash. 5. CRE validates: receipt authentic, price feed fresh, payment matches oracle rate. Triggers BOLT distribution to worker. 6. Coordinator posts reputation feedback to ERC-8004 on Monad. Future jobs route to proven workers first. Every job: priced by oracle, verified by oracle, settled by oracle.

Live infrastructure (deployed, working today): - Full E2E compute flow: agent submits job, coordinator routes, worker processes real LLM inference, ed25519 receipt returned - x402 HTTP micropayments on both Solana (SPL) and Monad (EVM) - BOLT SPL token on Solana, wBOLT ERC-20 on Monad - ERC-8004 identity and reputation registries on Monad testnet - Reputation-aware routing: higher score = priority matching - Three routing policies: FAST, CHEAP, PRIVATE - Desktop workers (Node.js + Ollama) running real inference - Mobile workers (React Native + Expo), approved on the Solana dApp Store - Real-time dashboard with job history and explorer links - 16,000+ lines TypeScript, 17 packages, MIT licensed Chainlink CRE addition (design, not yet deployed): - Price feed integration point identified in coordinator settlement pipeline - Demand-based pricing multiplier designed around queue depth and worker count - Oracle-verified settlement flow specified for CRE workflow execution
