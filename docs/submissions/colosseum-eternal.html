tagline
The compute marketplace where AI agents pay idle hardware per job, settled on Solana.
shortDescription
Dispatch is the first autonomous compute marketplace. AI agents send one HTTP request with payment, idle phones and desktops process the job, BOLT settles on Solana. No accounts, no API keys, no humans. 16,000+ lines of TypeScript, live on devnet, approved on the Solana dApp Store.
problemStatement
Every compute provider assumes a human operator. AI agents cannot create accounts, negotiate contracts, or manage API keys. Centralized providers (OpenAI, Anthropic): require human-provisioned API keys, enforce opaque rate limits, bill to credit cards. An agent cannot sign up. GPU rental networks (Akash, Render, Vast.ai): sell per-hour VM leases to developers who manually configure containers. Agents need per-job compute, not per-hour leases. Serverless inference (Replicate, Modal): better granularity, still centralized, still requires API keys, no verifiable proof that computation happened. The gap: no service lets an agent send one HTTP request with inline payment and get a verified result back. Agents do one thing well: HTTP requests. Build for that. Dispatch fills this. Per-job pricing from $0.001. HTTP-native payment via x402. Ed25519 signed receipts proving who computed what. The worker is an idle phone in someone's pocket.
whatItDoes
Dispatch is a two-sided compute marketplace. One side: AI agents that need cheap inference. Other side: idle phones and desktops that process it. An agent sends one HTTP POST with an x402 payment header (USDC). No SDK, no API key, no account. The coordinator scores workers by ERC-8004 reputation and BOLT stake, routes to the best match. The worker runs the job (LLM inference via Ollama on desktop, text processing on mobile), signs an ed25519 receipt over the output hash, returns the result. BOLT settles on Solana. Three routing policies: FAST (lowest latency), CHEAP (lowest cost), PRIVATE (paired device only). This is fully autonomous agent-to-agent compute. One agent pays another for work. Trust builds through onchain reputation. No human touches the transaction. Closest analogy: Helium for compute. The "hotspot" is a phone or laptop, the "coverage" is AI inference.
howItWorks
Six steps, fully autonomous: 1. Worker registers. Mints an ERC-8004 agent NFT on Monad testnet. Sets agentURI with capabilities and x402 support. Optionally stakes BOLT for priority. 2. Worker connects via WebSocket. Coordinator reads ERC-8004 reputation and stake tier. Atomic claiming prevents double-processing. 3. Agent submits a job. HTTP POST with x402 payment header (USDC). Coordinator scores workers: reputation (0-100) times stake multiplier (Open 1x, Verified 1.5x, Sentinel 2x). Highest wins. 4. Worker processes the job. Desktop: Ollama for LLM inference (summarization, classification, extraction). Mobile: text processing. Signs ed25519 receipt over SHA-256 output hash. 5. Settlement. x402 USDC auto-swaps to BOLT via Jupiter DEX. 5% protocol fee. 5% of BOLT burned permanently. Worker receives the rest. 6. Reputation update. Coordinator posts feedback to ERC-8004 Reputation Registry on Monad: score, skill tag, job type. Better reputation = more work.
solanaIntegration
Seven Solana primitives, all working: 1. BOLT SPL Token. Native Solana token (1B supply, 9 decimals). Real SPL transfers on devnet settle every job. 2. Mobile Wallet Adapter (MWA). Workers on Seeker authenticate via Solana MWA. Existing wallet, no new keys. 3. Jupiter DEX. x402 USDC auto-swaps to BOLT atomically during job commit. Fallback to direct distribution if DEX unavailable. 4. Ed25519 Signatures. Receipts use Solana's native signature scheme. Workers sign SHA-256 output hashes. Verifiable on-chain. 5. dApp Store. Release NFT minted, APK on Arweave. v1.4.0 approved. Search "Dispatch" on the Solana dApp Store. 6. Seeker-Native. Mobile app turns any idle phone into a worker node. Earn BOLT passively. 7. Staking Tiers. Open (0 BOLT), Verified (100 BOLT, 1.5x rep), Sentinel (1,000 BOLT, 2x rep, revenue share). 5% of every BOLT payment burned permanently.

businessModel
5% protocol fee on every job settlement. That is the entire revenue model. How it works: agents pay USDC via x402 HTTP headers. The coordinator auto-swaps USDC to BOLT via Jupiter DEX, deducts 5%, distributes the rest to the worker. Three value accrual mechanisms for BOLT: 1. Buy pressure. Every job triggers a USDC-to-BOLT swap on Jupiter. More jobs = more demand. Automatic, not speculative. 2. Supply lock. Workers stake BOLT for priority (Verified: 100 BOLT, 1.5x rep. Sentinel: 1,000 BOLT, 2x). Staked tokens leave circulation. 3. Burn. 5% of every BOLT payment burned permanently. Supply is strictly deflationary. At scale: 100K jobs/day at $0.005 avg = $25/day protocol revenue + $500/day BOLT buy pressure via Jupiter. No token required to participate. Workers earn from day one with zero stake. Protocol is open source (MIT). Value capture at the coordination layer.
