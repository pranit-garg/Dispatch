projectName
Dispatch
tagline
Multi-agent compute coordination where AI agents discover, pay, and verify each other.
shortDescription
Dispatch is a protocol where multiple AI agents coordinate autonomously. Client agents submit compute jobs over HTTP, the coordinator agent routes by reputation, and worker agents process real inference on idle hardware. All settlement and trust happen onchain with zero human involvement.
whatItDoes
Dispatch is a multi-agent system where three types of autonomous agents coordinate to process compute jobs. The requirement: "at least 2 AI agents working together." Dispatch has N+2: a client agent, a coordinator agent, and N worker agents running simultaneously. A client agent needs inference (summarization, classification, extraction). It sends one HTTP POST with an x402 payment header. No SDK, no API key, no account. The coordinator queries ERC-8004 reputation on Monad, scores workers by reputation and stake tier, routes to the best match. The worker runs the job via Ollama, signs an ed25519 receipt over the output hash, returns the result. Reputation feedback posts onchain. Each agent is autonomous: clients decide what to compute and how much to pay, the coordinator handles discovery and trust scoring, workers process jobs and build track records. No human touches any part of the flow.
multiAgentArchitecture
Three agent types form the A2A (agent-to-agent) system: 1. Client Agents: Any AI needing compute. HTTP with x402 payment. Selects routing (FAST, CHEAP, PRIVATE). Gets results with ed25519 receipts. 2. Coordinator Agent: Live worker registry. Queries ERC-8004 reputation on Monad. Composite scores (reputation x stake multiplier). Routes to optimal worker. Posts onchain feedback. 3. Worker Agents: Phones and desktops running inference via WebSocket. Atomic job claims. Ollama LLM inference. ed25519 receipts over SHA-256 hashes. Discovery: Workers mint ERC-8004 agent NFTs on Monad (skills, endpoint, payment support). Coordinator reads this registry. Clients need only the coordinator URL. Negotiation: x402 header carries the offer. Coordinator matches by routing policy. Pricing is protocol-defined. Settlement: BOLT on Solana (SPL), wBOLT on Monad (ERC-20). 5% fee, 5% burn. Jupiter DEX auto-swaps USDC to BOLT.
howItWorks
Step-by-step flow of a single job across all agents: 1. Workers register. Mint ERC-8004 agent NFT on Monad testnet (skills, endpoint, x402 support). Connect to coordinator via WebSocket. 2. Client submits a job. HTTP POST with x402 payment header (USDC). Specifies job type and routing (FAST/CHEAP/PRIVATE). 3. Coordinator scores workers. Reads ERC-8004 reputation onchain. Composite score: reputation (0-100) x stake tier (Open 1x, Verified 1.5x, Sentinel 2x). Selects highest scorer. 4. Worker claims and processes. Atomic claim prevents double-processing. Runs LLM inference via Ollama. Signs ed25519 receipt over SHA-256 output hash. 5. Settlement. USDC auto-swaps to BOLT via Jupiter DEX. Coordinator distributes BOLT to worker. 5% fee, 5% burned. 6. Reputation update. Coordinator posts to ERC-8004 Reputation Registry on Monad: score, skill tag, feedback hash. Worker record grows per completed job. All six steps happen without human input.
