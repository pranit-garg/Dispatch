shortDescription
Dispatch is a decentralized compute marketplace where AI agents pay for inference jobs using x402 HTTP micropayments. Two coordinators run in production: one on Solana (ExactSvmScheme) and one on Monad (ExactEvmScheme), both settling real USDC. x402 is not an add-on. It is how every job gets paid.
whatItDoes
Dispatch is a compute marketplace where idle devices run AI inference jobs and get paid through x402. A client sends an HTTP request to run a job. That request includes an x402 payment header with USDC authorization. The coordinator validates payment, routes the job to the best worker by reputation, and the worker processes it using a local LLM via Ollama. When the job completes, the worker signs an ed25519 receipt over the output hash. The coordinator verifies the receipt, settles USDC through x402, and posts reputation feedback on-chain via ERC-8004. x402 makes payment atomic with the HTTP request. No billing API, no invoice system, no payment portal. The agent includes payment in the request header, settlement happens in the response cycle. Two coordinators run on two chains. Solana uses ExactSvmScheme for SPL USDC. Monad uses ExactEvmScheme for EVM USDC. Same protocol, same job format, different settlement layers.
x402Integration
Dispatch implements x402 at the protocol level, not as a wrapper. Solana coordinator (port 4020): @x402/svm with ExactSvmScheme. Facilitator at x402.org, SPL USDC settlement on Solana devnet. Token resolution is automatic. Monad coordinator (port 4010): @x402/evm with ExactEvmScheme. Facilitator at x402-facilitator.molandak.org, EVM USDC settlement on Monad testnet. Both coordinators use the same pattern: 1. Create x402ResourceServer with HTTPFacilitatorClient 2. Register chain-specific scheme (ExactSvmScheme or ExactEvmScheme) 3. Apply paymentMiddleware to /job endpoint 4. Every job request must carry a valid x402 payment header 5. Validation and settlement happen inline with job processing The middleware intercepts each request, validates against the facilitator, and blocks unauthorized requests before any compute runs. Real x402 across two chain architectures (SVM and EVM), same interface. One HTTP header, two chains, real USDC.
howItWorks
End-to-end flow, focusing on the x402 payment path: 1. Client sends POST /job with an x402 payment header containing USDC authorization. 2. The paymentMiddleware validates the header against the chain-specific facilitator (x402.org for Solana, x402-facilitator.molandak.org for Monad). If payment fails, the request is rejected before any compute runs. 3. Coordinator scores all online workers by ERC-8004 on-chain reputation and routes the job. Three policies: FAST, CHEAP, PRIVATE. 4. The selected worker receives the job over WebSocket, runs LLM inference via Ollama, and signs an ed25519 receipt over the SHA-256 hash of the output. 5. Coordinator verifies the receipt, completes x402 USDC settlement through the facilitator, and posts reputation feedback to ERC-8004 contracts on Monad. The entire cycle (payment, compute, settlement) happens within a single HTTP request/response. No callbacks, no polling, no separate payment step.
