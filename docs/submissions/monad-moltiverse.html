Dispatch
tagline
The trust layer for autonomous agents on Monad.
shortDescription
Dispatch is the first A2A (agent-to-agent) compute marketplace with onchain trust. Agents buy inference from idle phones and desktops. Every job posts reputation feedback to ERC-8004 on Monad. Higher-reputation workers get priority. No humans in the loop.
whatItDoes
limit: 1000
Every compute marketplace today is human-mediated. Agents can't create accounts, negotiate contracts, or manage API keys. Dispatch removes humans entirely. An AI agent sends a single HTTP request with an x402 payment header. No SDK, no API key, no account. The coordinator checks the ERC-8004 registry on Monad, finds the best worker by reputation and stake, and routes the job. The worker processes it (real LLM inference via Ollama), signs an ed25519 receipt, and collects payment. The coordinator posts reputation feedback onchain. Done. This is agent-to-agent compute. One agent (the client) pays another agent (the worker) for a job. Trust builds automatically through ERC-8004 reputation on Monad. Workers are idle phones and desktops, not data centers. The result: a fully autonomous compute marketplace where agents discover, pay, and evaluate other agents with zero human intervention. Dispatch is the first project to use ERC-8004 as the backbone of a working reputation marketplace.
howItWorks
limit: 1000
Six steps, fully autonomous: 1. Worker registers onchain. Mints an ERC-8004 agent NFT on Monad. Sets an agent URI advertising capabilities (skills, endpoint, x402 support). 2. Worker connects to the coordinator via WebSocket. The coordinator reads the worker's ERC-8004 reputation score (async, non-blocking). 3. Client agent submits a job. HTTP POST with x402 payment header. The coordinator scores available workers: reputation (0-100) multiplied by stake tier (1x/1.5x/2x). Highest composite score wins. 4. Worker processes the job and signs an ed25519 receipt over the SHA-256 output hash. 5. Coordinator posts feedback to the ERC-8004 Reputation Registry on Monad. Score, skill tag, and job type are recorded onchain. 6. Reputation accumulates. Reliable workers build visible track records. Client agents query the registry directly to find trusted workers by skill and history.
erc8004Integration
limit: 1000
ERC-8004 is the core of Dispatch on Monad. Two deployed contracts, both live on testnet. Identity Registry (0x8004A818BFB912233c491871b3d84c89A494BD9e): Workers mint agent NFTs. Each NFT links to a registration file (offchain JSON) advertising skills, endpoint URL, and x402 payment support. Agents discover workers by reading the registry directly. Reputation Registry (0x8004B663056A597Dffe9eCcC1965A193B7388713): After every completed job, the coordinator posts onchain feedback: numeric score, skill tag, endpoint, and feedback hash. Workers accumulate per-skill reputation. Agents query this before routing. Reputation-aware routing (workerHub.ts) computes a composite score: base reputation (0-100) times a stake multiplier. Open tier (0 BOLT) = 1x. Verified (100 BOLT) = 1.5x. Sentinel (1,000 BOLT) = 2x. Higher score wins the job. wBOLT (ERC-20 on Monad) handles per-job payments via the WrappedBoltDistributor contract.
whyMonad
limit: 1000
Dispatch posts reputation feedback onchain after every job. Per-job trust, not batched trust. At 100K jobs/day, that is 100K onchain transactions. On Ethereum mainnet, roughly $50K daily in gas. On most L2s, still dollars per tx at scale. On Monad, pennies. Per-job reputation is the core mechanism. Without high throughput and low gas, it does not work. Batching weekly would destroy the real-time trust signal that makes the marketplace function. Three specific Monad advantages: 1. Fast finality: reputation updates visible to the next job request. A worker who just succeeded has that signal in seconds, not minutes. 2. Low gas: reputation can be granular. Per-skill, per-job-type scoring is practical. "llm-inference: 95/100 across 847 jobs" is more useful than "good worker." 3. EVM compatibility: ERC-8004 contracts deploy identically to Base, Ethereum, or any EVM chain. Workers register on Monad but the standard is portable.

techStack
TypeScript monorepo, 16,000+ lines, 17 packages, MIT licensed. - ERC-8004 SDK: packages/erc8004/, viem wrappers for Identity + Reputation registries on Monad - Coordinator: Express + WebSocket, reputation-aware routing, x402 payment validation - Workers: Desktop (Node.js + Ollama for LLM inference) + Mobile (React Native + Expo) - Payments: x402 HTTP micropayments (EVM ExactEvmScheme on Monad, SPL on Solana) - Tokens: wBOLT (ERC-20 on Monad) + BOLT (SPL on Solana) - Dual-chain: Monad handles identity, reputation, and wBOLT. Solana handles SPL settlement and dApp Store distribution.
